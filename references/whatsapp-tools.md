# WhatsApp 工具工作流  ## 适用场景 用户需要检查哪些手机号注册了 WhatsApp，或向潜在客户发送 WhatsApp 消息。  ## 环境搭建：mcp-whatsapp  ### 首次安装 1. 克隆并编译：    git clone https://github.com/Sealjay/mcp-whatsapp    cd mcp-whatsapp && make build 2. 启动守护进程：    ./bin/whatsapp-mcp serve 3. 浏览器打开 http://127.0.0.1:8765/pair 4. 用手机 WhatsApp 扫码（设置 > 已关联设备 > 关联设备） 5. 会话保存在 ./store/whatsapp.db（约 20 天后需重新认证）  ### 验证连接状态 使用 mcp-whatsapp get_status 检查是否已连接。  ## 工作流一：批量验证 WhatsApp 号码  当用户说："检查这些号码有没有 WhatsApp" 或 "哪些号码有 WhatsApp"  1. 收集电话号码（必须含国际区号，见下方区号表） 2. 格式标准化：去除空格、短横线，确保 +国家代码 开头 3. 使用 mcp-whatsapp is_on_whatsapp 批量查询 4. 返回：哪些号码已注册 WhatsApp 5. 呈现结果：WhatsApp 活跃 X 个 / 未注册 Y 个 / 无效 Z 个 6. 在 Snov.io 列表中标记 WhatsApp 活跃状态  ## 工作流二：发送 WhatsApp 开发信  当用户说："给这些联系人发 WhatsApp 消息"  1. 向用户确认消息模板（或使用下方默认模板） 2. 使用 mcp-whatsapp send_message 逐个发送 3. 记录发送状态 4. **每条消息间隔至少 6 秒**，避免触发 WhatsApp 风控  默认开发信模板（可与用户确认后修改）： "您好 [姓名]，我是 [公司] 的 [你的名字]。 我们专注于 [产品/服务]。 注意到贵司 [个性化备注]。方便简单聊一下吗？ 此致，[你的名字]"  ## 工作流三：邮件 + WhatsApp 组合跟进  1. 通过 Snov.io 查找邮箱（见 email-finding.md） 2. 验证邮箱（见 email-verification.md） 3. 从 Snov.io 结果中提取电话号码 4. 用 is_on_whatsapp 检查 WhatsApp 状态 5. 先发邮件，等 2 天 6. 对未回复的联系人发 WhatsApp 跟进  ## 全球主要贸易国别区号  ### 北美 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 美国 | +1 | 极高 | | 加拿大 | +1 | 极高 | | 墨西哥 | +52 | 极高 |  ### 南美 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 巴西 | +55 | 极高 | | 阿根廷 | +54 | 极高 | | 哥伦比亚 | +57 | 极高 | | 智利 | +56 | 极高 | | 秘鲁 | +51 | 极高 | | 厄瓜多尔 | +593 | 高 | | 委内瑞拉 | +58 | 高 |  ### 欧洲 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 英国 | +44 | 极高 | | 德国 | +49 | 极高 | | 法国 | +33 | 高 | | 意大利 | +39 | 极高 | | 西班牙 | +34 | 极高 | | 荷兰 | +31 | 极高 | | 比利时 | +32 | 高 | | 波兰 | +48 | 高 | | 葡萄牙 | +351 | 极高 | | 瑞典 | +46 | 高 | | 瑞士 | +41 | 高 | | 奥地利 | +43 | 高 | | 爱尔兰 | +353 | 高 | | 希腊 | +30 | 高 | | 捷克 | +420 | 中 | | 罗马尼亚 | +40 | 中 | | 匈牙利 | +36 | 中 |  ### 亚洲（仅发达国家） | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 日本 | +81 | 低（LINE 为主） | | 韩国 | +82 | 低（KakaoTalk 为主） | | 新加坡 | +65 | 极高 |  ### 中东 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 阿联酋 | +971 | 极高 | | 沙特阿拉伯 | +966 | 极高 | | 卡塔尔 | +974 | 极高 | | 科威特 | +965 | 极高 | | 阿曼 | +968 | 高 | | 巴林 | +973 | 极高 | | 约旦 | +962 | 高 | | 埃及 | +20 | 高 | | 土耳其 | +90 | 极高 | | 以色列 | +972 | 极高 | | 黎巴嫩 | +961 | 高 | | 伊拉克 | +964 | 中 | | 伊朗 | +98 | 中 |  ### 大洋洲 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 澳大利亚 | +61 | 极高 | | 新西兰 | +64 | 极高 |  ### 其他发达国家 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 挪威 | +47 | 高 | | 丹麦 | +45 | 高 | | 芬兰 | +358 | 高 | | 冰岛 | +354 | 高 | | 卢森堡 | +352 | 高 | | 摩纳哥 | +377 | 高 | | 安道尔 | +376 | 高 | | 列支敦士登 | +423 | 高 |  ### 中美洲及加勒比 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 哥斯达黎加 | +506 | 极高 | | 巴拿马 | +507 | 极高 | | 多米尼加 | +1-809 | 极高 |  ### 非洲 | 国家 | 区号 | WhatsApp 渗透率 | |------|------|----------------| | 南非 | +27 | 极高 | | 尼日利亚 | +234 | 极高 | | 肯尼亚 | +254 | 极高 |  ## 注意事项 - 仅限个人 WhatsApp 账号（非 Business API） - 约 20 天需重新扫码认证 - WhatsApp 有频率限制，**每条消息间隔至少 6 秒** - 不支持批量群发 API，需逐条发送并添加延迟 - 同一号码 24 小时内首次发消息建议先验证 WhatsApp 活跃状态  ## 常见错误 & 自救  | 错误 | 原因 | 解决 | |------|------|------| | mcp-whatsapp 连不上 | 守护进程未启动 | cd ~/mcp-whatsapp && ./bin/whatsapp-mcp serve | | 扫码失败 | 20 天会话过期 | 重新打开 http://127.0.0.1:8765/pair 扫码 | | is_on_whatsapp 返回空 | 号码格式错误 | 确保号码以 +国家代码 开头，例：+12015551234 | | send_message 被限 | 发送频率过快 | 每条消息间隔 >= 6 秒，24h 内不超过 100 条 | | WhatsApp 账号被封 | 大量群发检测 | 减少日发送量，增加消息个性化内容 | | 手机离线 | WhatsApp Web 要求手机在线 | 保持手机 WhatsApp 连接网络 | | 日本/韩国号码无 WhatsApp | LINE/KakaoTalk 为主 | 换用邮件联系 |
