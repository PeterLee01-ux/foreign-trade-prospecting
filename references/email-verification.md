# 邮箱验证工作流  ## 适用场景 用户有一个邮箱列表，需要验证哪些是有效的、可投递的。  ## 环境搭建：check-if-email-exists（本地 Docker）  ### 首次部署 ```bash docker run -d --name reacher -p 8080:8080 --restart unless-stopped reacherhq/backend:latest ```  ### 验证服务是否运行 ```bash curl http://localhost:8080/health ```  或 Python 脚本： ```bash python scripts/verify_email_direct.py --health ```  ## 三种验证方式（按推荐顺序）  ### 方式一：Reacher Docker — 本地、无限制、免费 ```bash # 单个验证 python scripts/verify_email_direct.py test@example.com  # 批量验证 python scripts/verify_email_direct.py --batch emails.txt --output results.csv  # 启动 HTTP 代理服务（端口 8081） python scripts/verify_email_direct.py --server ```  返回字段说明： - is_reachable：valid（有效）| invalid（无效）| risky（有风险）| unknown（未知） - mx_valid：是否找到 MX 邮件交换记录 - smtp_valid：SMTP 握手结果 - syntax_valid：格式是否合法 - has_catch_all：全域接受 - is_disposable：一次性邮箱 - is_role_account：公共邮箱（info@/sales@）  ### 方式二：Prospector MCP（免费版每天 50 次） 单个验证：prospector verify_email 批量验证（最多 25 个）：prospector verify_emails_batch  返回：status（valid/invalid/unknown）、score（0-100）、mx_host、catch_all  ### 方式三：Snov.io MCP / API（付费额度） ```bash # MCP 模式 snovio_verify_emails  # 免 MCP 模式 python scripts/snovio_api.py verify test@acme.com python scripts/snovio_api.py verify-batch email1@a.com email2@b.com ```  ## 批量验证流程  1. 将所有待验证邮箱汇入列表 2. 数量不超过 50 个：优先用 prospector verify_emails_batch（免费） 3. 数量超过 50 个：用 `python scripts/verify_email_direct.py --batch emails.txt`（本地无限制） 4. 需要详细投递数据分析：用 Snov.io  ## 结果解读  | 状态 | 处理方式 | |------|---------| | valid / reachable | 安全，可直接发送 | | risky | 可能退信，重要客户可尝试 | | invalid / unreachable | 从列表中删除 | | catch_all | 域名接受所有邮件，谨慎测试 | | disposable | 一次性临时邮箱，删除 | | role_account | info@/sales@ 类公共邮箱，参与度低，相关则保留 |  ## 验证完成后  1. 在 Snov.io 列表中标记已验证邮箱 2. 计算有效率 = 有效数 / 总数 3. 向用户报告结果摘要 4. 建议下一步：对有效邮箱开始开发，对无效邮箱重新搜索  ## 常见错误 & 自救  | 错误 | 原因 | 解决 | |------|------|------| | Reacher 连不上 | Docker 未运行 | `docker run -d -p 8080:8080 --name reacher reacherhq/backend:latest` | | Docker 报权限错 | 非管理员 | 加入 docker 用户组或 sudo | | Snov.io 403 | API Key 无效或过期 | 让用户去 Snov.io 重新获取 | | Snov.io 429 | 请求频率超限 | 等待 60 秒后重试 | | Prospector 报错 | Node.js 未安装 | 检查：node --version，需 >=18 | | 免 MCP 模式密钥错 | env var 未设 | export SNOVIO_CLIENT_ID/SECRET |
